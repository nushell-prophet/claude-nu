# Initial request (only the user can edit this section)

Audit on 2026-05-10 against Claude Code 2.1.138 found that `parse-session` and
its helpers silently miss data introduced by recent Claude Code releases. Fix
the parser so that real recent sessions yield correct values for the fields
the command already advertises.

# LLM agent updates section

## Findings (full reproducers in conversation transcript 2026-05-10)

All bugs verified against real sessions in `~/.claude/projects/`. Unit tests
(79/79 passing) do not catch any of these because `tests/fixtures/sessions/`
is empty.

### BROKEN

1. **Tool rename `Task` → `Agent` not tracked.**
   `extract-agents` (`claude-nu/commands.nu:338`) filters
   `name? == "Task"`. Real sessions in 2.1.138 use `Agent`, `TaskCreate`,
   `TaskStop`, `TaskUpdate`. Result: `agents` and `agent_count` are always
   `[]` / `0` for any session that ran sub-agents.
   Same filter is repeated in `extract-tool-stats` (commands.nu:358–360).

2. **`extract-session-metadata` reads "first non-summary record".**
   In 2.1.138 the first non-summary record is now `permission-mode` or
   `file-history-snapshot`, neither of which carries `cwd` / `version` /
   `gitBranch` / `slug`. `permission-mode` carries only `sessionId`. Result:
   for projects whose first record is `file-history-snapshot` (e.g. cozy,
   dotfiles), every metadata field including `session_id` returns `""`.
   Fix direction: walk records and pick the first one that actually carries
   each field.

3. **`Skill` / `EnterPlanMode` tool detection is stale.**
   Plan mode is now a top-level `permission-mode` record with
   `permissionMode: "plan"`, not derived from a `EnterPlanMode` tool call.
   `Skill` invocations no longer appear in the tool stream that the parser
   consumes — confirm whether the tool was renamed or moved.

### GAP (silently missed data)

4. **New top-level record types ignored entirely:**
   `ai-title`, `last-prompt`, `permission-mode`, `queue-operation`,
   `attachment`, `file-history-snapshot`, `agent-name`. Currently the parser
   only consumes `summary`, `user`, `assistant`.
   - `ai-title.aiTitle` is the canonical short summary in new sessions; old
     `summary` records are mostly absent. `summary` field is empty in every
     recently-sampled session. Use `ai-title.aiTitle` as fallback when no
     `summary` record present.
   - `permission-mode.permissionMode` (`auto` / `bypassPermissions` / `plan`
     / `default`) supersedes `EnterPlanMode` tool detection.
   - `last-prompt.lastPrompt` — most recent user prompt as top-level field.

5. **New tool names absent from stats:**
   `Agent`, `Monitor`, `ToolSearch`, `TaskCreate`, `TaskStop`, `TaskUpdate`,
   `AskUserQuestion`. None are counted anywhere.

## Completed

Five vendored fixtures cover: permission-mode-first (with plan mode),
file-history-snapshot-first (with TaskCreate/Update/Stop), and a
legacy 2.1.129 user-first session. Tests went from 79 to 97 passing.

Fixes shipped (each its own commit):

- `extract-agents` accepts `Task` and `Agent` (BC kept). TaskCreate/
  Update/Stop intentionally NOT in this list — they're TODO-list ops
  with different schema, counted separately as tool stats.
- `extract-session-metadata` walks records to find each metadata
  field instead of reading "first non-summary record". Signature
  changed from `record -> record` to `table -> record`; not
  re-exported via `mod.nu` so no public break.
- `extract-summary` helper added: prefers `summary` record, falls
  back to `ai-title.aiTitle`. Used at all three call sites.
- `plan_mode_used` derived from `permission-mode.permissionMode ==
  "plan"` OR legacy `EnterPlanMode` tool call (either signal). The
  bool column kept for BC.
- New count columns: `task_create_count`, `task_update_count`,
  `task_stop_count`, `monitor_count`, `tool_search_count`. Matching
  flags on `parse-session`; included under `--all`.

Deferred (needs decision):

- `last-prompt`, `queue-operation`, `attachment`, `agent-name`,
  `file-history-snapshot` record types still ignored. The audit
  flagged them as silently missed but it's unclear which deserve
  promotion to first-class columns vs being just metadata noise.
  Skipped per the "no new top-level columns yet" instruction.
- `Skill` invocations: audit said they no longer appear in the tool
  stream. Existing `skill_invocations` column still counts old `Skill`
  tool calls; whether the new shape lives elsewhere is unknown.
  Not changed.
- Subagent JSONLs under `<sid>/subagents/agent-*.jsonl` are still
  not enumerated by `sessions`/`messages`. Tracked in a separate
  todo (20260510-1959-sessions-enumeration.md).
- Thinking-only assistant turns and `tool_result` nested-table
  rendering: tracked in 20260510-2002-export-session-rendering.md.
